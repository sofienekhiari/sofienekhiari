Instructions for Your Response

The following steps must all be executed in full for every single prompt, with no steps skipped, merged, or summarized. Treat each step as mandatory and independent; do not omit any part of a step even if the topic seems simple.

Please follow these steps carefully:

Step 1: Generate Scientific Explanation in Quarto Markdown Format

Purpose: This explanation will be used for a scientific tutorial/review publication on my Quarto-based academic website. The content must be of publishable quality suitable for an academic journal or comprehensive review article.

Requirements: - Write 3-6 relatively short paragraphs (aim for concise blocks rather than long walls of text; overall length typically 300-800 words total) - Use scientifically rigorous yet accessible language - maintain academic standards while remaining understandable to readers new to the topic (intelligent beginners / non‑experts) - Provide clear, accurate explanations with examples where helpful - Make the explanation contextually relevant to my broader learning plan shown above - Output directly in Quarto markdown (.qmd) format with proper citation commands

Writing Standards: - Use precise scientific terminology with clear definitions - Structure paragraphs logically with clear topic sentences - Maintain formal academic tone appropriate for publication - Include relevant examples, mechanisms, or applications where appropriate - Ensure claims are well-supported and accurately cited

Citation Requirements - CRITICAL: - ONLY cite open-access papers or papers with publicly available PDFs - If you cannot access the full text of a source, do not cite it - Prioritize academic sources: peer-reviewed papers, textbooks, academic reviews - Use general websites/blog posts ONLY when academic sources are unavailable for that specific information - Use Quarto/Pandoc citation format: [@citationkey] for parenthetical citations or @citationkey for in-text citations - Multiple citations should use: [@key1; @key2; @key3] - Create meaningful citation keys (e.g., smith2023protein, jones2022review) - Citations must be in square brackets with @ symbol, following Pandoc markdown syntax

Quarto Markdown Formatting:

# Use standard Markdown paragraph formatting
# Use **bold** for emphasis on key terms
# Use *italics* for species names or emphasis
# Use `code` for technical notation if needed
# Citations: [@citationkey] or @citationkey for in-text

Publication Quality Criteria - MUST FOLLOW:

  1. Term standardization: Consistently use technical terms and nomenclature throughout the text; define preferred terminology at first use and apply the same terms uniformly to avoid confusion or ambiguity.

  2. Abbreviation management: Introduce abbreviations only after spelling out their full form, e.g., “quantitative structure–activity relationship (QSAR),” and reuse abbreviations consistently for clarity.

  3. Symbol consistency: Use mathematical and scientific symbols uniformly for specific quantities or entities throughout the document; avoid changing notation mid-text and clarify all symbols if needed.

  4. Primary source preference: Prioritize taking factual claims and data directly from original research articles or technical reports, rather than reviews or tertiary sources, unless no primary source is available.

  5. No orphan claims: Ensure every non-trivial factual statement or data point is supported by a specific citation; check that no claim is left without evidence especially in technical or novel areas.

  6. No citation chaining without verification: Do not cite a secondary source for a fact purportedly from another paper without confirming and directly citing the original source for accuracy and transparency.

  7. Logical transitions: Strategically use connecting phrases (“Furthermore,” “However,” “As a consequence,” etc.) to transition coherently between different concepts or paragraphs, enhancing readability and scientific clarity.

  8. Progressive complexity: Arrange subject matter so the presentation proceeds from fundamental concepts to increasingly advanced topics, ensuring readers are not lost due to abrupt complexity jumps.

  9. Avoid unnecessary jargon: Use simple, understandable language wherever possible without sacrificing scientific accuracy; introduce technical jargon only when necessary with adequate explanation.

  10. Figure references in text: Clearly reference all suggested figures in the appropriate place within the text (e.g., “(see Figure 2)”), so that readers know when to consult visual information for clarification.

  11. Figure caption guidance: Provide explicit suggestions for figure captions, describing what information should be included to contextualize figures and how they relate to the discussed concepts.

  12. Avoid oversimplification: While making complex topics accessible, ensure that no critical nuances, caveats, or limitations are omitted; provide full context even when summarizing.

  13. Internal consistency: Throughout the publication, maintain agreement in terminologies, symbol use, and factual claims; ensure that statements in one part do not contradict information elsewhere in the document.

  14. Current state clarity: Clearly indicate in the text what is widely accepted, what is cutting-edge or under recent development, and what remains controversial or unexplored, to avoid confusing readers about the status of the science.

  15. Research gap identification: Actively point out areas where further investigation is needed, what open questions remain, and where consensus is lacking, enabling readers to understand and engage with ongoing research challenges.

  16. Critical evaluation: Do not merely recount established findings; appraise the strength, limitations, and implications of existing evidence, and provide a balanced, critical perspective on the literature.

Step 4: Verify Each Citation with Confidence & Plagiarism Scores

This is critical: For each citation you provided, you must perform comprehensive verification:

Core Verification Requirements: 1. Access or recall the actual source content 2. Quote the exact text or section from the source that supports your claim 3. Provide a confidence score (0.0-1.0) indicating certainty that the source contains the claimed information 4. Provide a plagiarism score (0.0-1.0) indicating how closely your text matches the original

Additional Quality Checks - MUST PERFORM:

  1. Cross-reference multiple sources: Validate each key claim by checking for its presence across at least two independent sources whenever possible, aiming for triangulation to ensure consensus and to avoid reliance on potentially anomalous or isolated findings.

  2. Date relevance check: Confirm all cited sources are appropriately current, especially in fast-evolving fields, by verifying their publication date and ensuring outdated sources are flagged or avoided unless they are foundational to the topic.

  3. DOI/URL validation: Ensure each cited reference includes a functioning DOI or URL, confirming that the citation resolves and leads to the full source text, thereby guaranteeing accessibility and traceability for verification.

  4. Citation context check: Examine the context in which the information appears in the cited source to make sure claims are not quoted out of context or misinterpreted; verify surrounding text and section to ensure the claim is represented accurately.

  5. Phrase-level comparison: Systematically compare generated content with the source material at the phrase or sentence level to detect any direct copying of five or more consecutive words, and assess for originality.

  6. Technical term density: Check if technical vocabulary and domain-specific terms are overused, used inconsistently, or match the source precisely; significant matches should prompt further scrutiny for potential plagiarism.

  7. Paraphrase quality assessment: Evaluate the degree of paraphrasing in each sentence by analyzing the structure, word choices, and grammar relative to the original source, ensuring active rewording and not mere synonym substitution.

  8. Quote identification: Distinguish between content that should be paraphrased and statements or definitions that are better rendered as direct quotes; explicitly mark such passages and properly format quotations.

  9. Logical flow check: Assess paragraph and argument structure so each sentence logically follows from the previous, with clear progression of thought; verify that concepts are introduced in an orderly fashion and transitions between ideas are fluent.

  10. Completeness check: Review the section to ensure all key subtopics and points identified in the learning plan are covered, and that no major aspect or requirement is omitted from the explanation.

  11. Accuracy cross-validation: Compare each major factual claim with several reliable, independent sources to confirm accuracy and consistency, flagging any claims that vary or are contradicted elsewhere.

  12. Depth appropriateness: Ensure the level of detail matches the intended scope and audience—provide sufficient explanation without excessive superficiality or unwarranted tangents, in line with scientific tutorials or reviews.

  13. Definition verification: Check that all technical terms, abbreviations, and symbols are correctly defined on their first appearance, and that definitions are consistent with authoritative sources.

  14. Claim strength matching: Ensure the language used to express claims (such as “suggests,” “demonstrates,” or “proves”) matches the level of evidence provided in the cited sources, avoiding overstatement or unwarranted certainty.

  15. Verification confidence scoring: For each claim and citation, assign a confidence score reflecting the certainty that the source contains the claimed information, considering access, specificity, and context.

  16. Missing citation identification: Flag statements and factual claims that lack citations but ought to be supported, especially where domain knowledge or data is not common sense or general knowledge.

  17. Over-citation check: Ensure that citations are only used where necessary (for non-trivial claims) and avoid citing trivial information or common scientific knowledge, which can clutter the publication.

  18. Citation recency distribution: Track and report the temporal range of citations used, ensuring that the explanation includes a balanced mixture of both recent research and foundational work relevant to the field.

Score Definitions: - Confidence Score: - 1.0 = Absolutely certain; can quote exact supporting text - 0.7-0.9 = High confidence; source clearly supports claim - 0.4-0.6 = Moderate confidence; source likely supports claim - 0.0-0.3 = Low confidence; uncertain if source contains this information

  • Plagiarism Score:
    • 0.0 = Completely original paraphrasing; no matching phrases
    • 0.1-0.3 = Well paraphrased; minimal similarity
    • 0.4-0.6 = Moderate paraphrasing; some similar phrases
    • 0.7-0.9 = Poor paraphrasing; many matching phrases
    • 1.0 = Direct copy; verbatim text from source

Step 5: Automated Post-Processing Verification

After generating the content and performing manual citation verifications, perform the following automated checks for each citation:

  1. DOI Resolution Check: Verify that the DOI resolves properly using the CrossRef API or DOI system. Confirm the DOI link is functional and leads to the correct article.

  2. Open-Access Verification: Check if the cited paper is open-access. If the paper is NOT open-access or does not have a publicly available PDF, FLAG this citation immediately.

  3. Full-Text PDF Access: If the paper is confirmed as open-access, attempt to download the PDF from the provided URL or DOI.

  4. Content Verification in PDF: Search the downloaded PDF for the exact quoted supporting text using:

    • Direct string search for exact phrases
    • Semantic similarity search for paraphrased content
    • Verify the location (page number, section) matches what was reported

  5. Flagging Protocol: Create a comprehensive list of all citations that fail ANY of these checks:

    • DOI does not resolve
    • Paper is not open-access
    • PDF cannot be downloaded
    • Quoted text cannot be found in the PDF
    • Location information is incorrect

Implementation Notes for Using This Template

Setting Up Your Quarto Document

Your final Quarto document should have this structure:

YAML Header (in your .qmd file):

---
title: "Your Article Title"
author: "Your Name"
date: today
bibliography: references.bib
csl: nature.csl  # or your preferred citation style
format:
  html:
    toc: true
    toc-depth: 3
---

Bibliography File (references.bib): - Create a file named references.bib in the same directory as your .qmd file - Paste all BibTeX entries provided by the agent into this file

Citation Style (optional): - Download a .csl file for your preferred citation style from the Zotero Style Repository - Place it in your project directory and reference it in the YAML header

Workflow:

  1. Fill in the template placeholders with your learning plan and specific topic
  2. Submit the complete prompt to your LLM agent (Perplexity Labs)
  3. Receive the generated Quarto markdown content
  4. Copy the markdown content into your .qmd file
  5. Copy the BibTeX entries into your references.bib file
  6. Review only the flagged citations manually
  7. Render your Quarto document with quarto render yourfile.qmd
  8. Publish to your Quarto website

Notes for Future Iterations

  • Default to 2-3 paragraphs; use 4 paragraphs for complex topics requiring more depth
  • Request specific figure types if you have preferences (e.g., “prioritize schematic diagrams”)
  • Specify citation style (CSL file) if you have a preference beyond the default Chicago style
  • Adjust confidence score thresholds if you want stricter verification (e.g., only accept scores > 0.8)
  • Can request specific Quarto extensions or formatting conventions
  • For website publication, ensure all figures are properly licensed for web use
